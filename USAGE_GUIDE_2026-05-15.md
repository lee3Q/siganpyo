# 시간표 (SiGanPyo) 사용법 가이드

> 이상규용. 매일 아침 폰으로 열어서 확인.
> 주소: https://lee3q.github.io/siganpyo/

---

## 1. 접속

브라우저에서 `https://lee3q.github.io/siganpyo/` 접속.
- 폰 홈화면에 추가 가능 (PWA). Safari→공유→홈 화면에 추가.
- 오프라인에서도 열림 (Service Worker 캐시).

## 2. 매일 아침 확인

오늘 날짜의 블록이 자동으로 표시됨.

| 항목 | 설명 |
|------|------|
| 시간 블록 | 오늘 해야 할 일이 시간 순으로 나열 |
| 카테고리 색상 | 루틴=초록, 노무사=보라, 휴식=파랑 등 |
| 체크리스트 | 각 블록에 [ ] 항목. 클릭하면 체크 |
| advice | 블록 하단의 조언. 읽기만 하면 됨 |

## 3. 데이터 구조

```
~/시간표/data/
├── 2026-05-16.json   ← D-7 (토)
├── 2026-05-17.json   ← D-6 (일)
├── ...
└── 2026-05-23.json   ← D-day (토)
```

각 파일 구조:
```json
{
  "date": "2026-05-16",
  "blocks": [
    {
      "id": "d7-001",
      "startTime": "07:30",
      "endTime": "08:00",
      "title": "기상 + 아침 루틴",
      "category": "루틴",
      "color": "green",
      "notes": "설명",
      "advice": "조언",
      "checklist": [
        { "text": "할 일", "done": false }
      ],
      "status": "planned",
      "source": "ai"
    }
  ]
}
```

## 4. 데이터 수정이 필요할 때

### 방법 A: Claude에게 말하기

```
시간표 5월 17일에 "운동 30분" 블록 추가해줘. 18:00~18:30.
```

Claude가 JSON 파일 수정 → git push → 자동 배포.

### 방법 B: 직접 수정

1. `~/시간표/data/2026-05-17.json` 열기
2. `blocks` 배열에 새 객체 추가 (위 구조 참조)
3. `cd ~/시간표 && git add . && git commit -m "수정" && git push`

push 후 1~2분 내 GitHub Pages에 반영.

## 5. 시험 기간 (5/16~5/23) 사용법

| 날짜 | D-N | 포커스 과목 |
|------|-----|------------|
| 5/16(토) | D-7 | 기출 마무리 + 전과목 점검 |
| 5/17(일) | D-6 | 노동법1 파이널 |
| 5/18(월) | D-5 | 노동법2 + 민법 |
| 5/19(화) | D-4 | 민법 마무리 + 오답 |
| 5/20(수) | D-3 | 사회보험법 + 경영학 |
| 5/21(목) | D-2 | 전과목 + 모의고사 4회 |
| 5/22(금) | D-1 | 포켓카드 + 숫자 점검 |
| 5/23(토) | D-day | 시험 + 유튜브 런칭 |

각 날짜의 JSON에 학습 계획이 이미 들어있음.
아침에 PWA 열어서 해당 날짜 확인 → 체크리스트 따라 진행.

## 6. 문제 발생 시

| 증상 | 해결 |
|------|------|
| 오늘 날짜가 안 보임 | 브라우저 새로고침 (당겨서 리프레시) |
| 오프라인에서 안 열림 | 한 번 온라인에서 열면 캐시됨 |
| 데이터가 안 바뀜 | git push 후 1~2분 대기 후 새로고침 |
| 폰에서 레이아웃 깨짐 | 세로 모드로 확인. 가로는 미지원 |
